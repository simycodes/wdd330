Local Storage

Local storage in JS applications is used in remembering the userâ€™s information or to keep track of the changes that a user makes to a JS web page so that these changes can be there again when the user access the web app again after the app has been closed or stopped.Usually when an application has been stopped all that it was working on is gone without being stored unless if the application has mechanism in place to save all the changes it has been.

Storing user data

With JS applications there is an inbuilt object called localstorage that is used to store user data without having to connect to databases such as mongo database or relational databases. All the data is stored by localStorage object and is attached to the browser that was used by the user when inserting or changing data on the JS application.

With this mechanism in place, a user may access the JS application any other time even after closing the browser and he will find the data and changes he made just like they were before closing the application.
The localStorage object has 2 main methods that it uses to store JS app data and to also retrieve the data.

-- setItem() method is used to store user data locally.
-- getItem() method is used to retrieve the data stored in the JS local storage.

NOTE: The setItem accepts two arguments (a pair), the key (first argument) and the corresponding value (second argument). You use the key to get access to the value that needs to be used/displayed.

Example of storing user data using LocalStorage object
localStorage.setItem(storyName, storyHTML);

Example of accessing and displaying user data using LocalStorage object
var storyHTML = localStorage.getItem(storyName);
console.log(storyHTML)
