Objects, Document Object Model(DOM) and Events

Using objects

Objects are usually created to represent entities of the real world, like users and orders.
An object in JavaScript is a self-contained set of related values and functions. Objects are often used to keep any related information and functionality together in the same place.

In JS there are six primitive data types (strings, numbers, booleans, symbols, undefined, and null) and the rest are referred to as objects.
These include arrays (inbuilt object), functions, object literals

Object Literals/Objects

Object literals/Objects are a distinguishing feature of the JavaScript language, as they allow objects to be created quickly without the need for defining a class. Objects can inherit properties from other objects. 

                        Example of an Object

                        const superman = {
                            name: 'Superman',
                            realName: 'Clark Kent',
                            height: 75,
                            weight: 235,
                            hero: true,
                            villain: false,
                            allies: ['Batman','Supergirl','Superboy'],
                            fly() {
                                return 'Up, up and away!';
                            }
                        };

Objects are accessed using the Dot notation(more common) and bracket notation.

Objects are assigned by reference. This means that if a variable is assigned to an object that already exists, it will simply point to the exact same space in memory. So any changes made using either reference will affect the same object. 

                        const fruit = { 
                            name: 'Banana' 
                        };

                        const smallFruit = fruit;

                        fruit and smallFruit are basically the same.Changing the name 
                        property (Banana) of fruit to Apple will make the name property
                        of smallFruit to become Apple as well.Same will occur if changes
                        are made in smallFruit.

Document Object Model (DOM)

The Document Object Model, or DOM for short, represents an HTML document as a network of connected nodes that form a tree-like structure. The HTML tag is the root node, and every other part of the document is a child node of this.

The Document Object Model (DOM) allows you to access elements of a web page and enable interaction with the page by adding and removing elements, changing the order, content and attributes of elements, and even altering how they are styled.

DOM involves the interaction with elements of a web page/document.It is a way in which JS talks with HTML elements.It has the capacity to create, access, add, remove and change the order of these HTML elements.
It also has the capacity to change the content(InnerHTML of elements), attributes and the CSS of these HTML elements (by adding & removing classes and ids on elements).
Its a great tool that is used to make and improve user interactions with a web page.

We can use JavaScript to access and modify different parts of a web page using a special built-in object called document.

Getting Elements from web page

Methods that access elements return a node object or a node list, which is an array-like object. These objects can then be assigned to a variable and be modified or inspected.

Node lists are array-like objects, but they are not arrays. You can access each item using index notation. For example, document.images[0] will return the first image in the node list of all the images in the document.
They also have a length property( document.images.length ) , which can be used to iterate through every element using a for loop.
for (let i=0 ; i < document.images.length ; i++) { }

The main methods used to access elements include: getElementById(), getElementsByTagName(), getElementsByClassName(), querySelector(), querySelectorAll()

childNodes and Children Properties

The childNodes property is a list of all the nodes that are children of the node concerned.
childNodes property returns all the nodes that are children of an element. This will include any text nodes, and since whitespace is treated as a text node, there will often be empty text nodes in this collection.
The children property only returns any element nodes that are children of that node, so will ignore any text nodes.

Finding the Value of a Node

The value of a node (element) can be found using the textContent And innerHTML properties.

Getting and Setting Attributes

All HTML elements have a large number of possible attributes such as 'class', 'id', src, and 'href'. The DOM has numerous getter and setter methods that can be used to view, add, remove or modify the value of any of these attributes.

These methods include,getAttribute('-attributeName-'), setAttribute('-attributeName-',-attributeValue-)

classList Property and toggle method

The classList property is a list of all the classes an element has. Its add method can be used to add a class to an element without overwriting any classes that already exist and its remove() method can easily remove a class from an element.

The toggle() method is a very useful method that will add a class if an element does not have it already, and remove the class if it does have it. It returns true if the class was added and false if it was removed. 

        heroes4.classList.toggle('hero'); // will add the 'hero' class
        console.log(heroes4.className);

        heroes4.classList.toggle('hero'); // will remove the 'hero' class
        console.log(heroes4.className);

the document.getElementsByClassName() and document.getElementsByTagName() methods are live collections that will update to reflect any changes on the page.
Hence can be useful when creating nav bars that change when clicked on.

Event Listeners

Events are another part of the DOM. They provide a link (or an interface) between the web page and user interactions.
Event Listeners are what users use/interact with to see how interactive a webpage/website is.Its what programmers use to add attach dynamic changes and these come inform of clicks or pressing a key.

Event listeners are added to elements on the page and are part of the DOM. The recommended way of dealing with events, and the current standard, is to use event listeners.
The addEventListener() method is used to attach an event to a DOM element. 

    document.body.addEventListener('click',doSomething);

    Its first parameter is the type of event, and 
    the second is a callback function that is invoked when the event occur

    Note that the parentheses are not placed after the function when it's used as the 
    argument to an event listener -doSomething- no (), otherwise, the function will 
    actually be called when the event listener is set, instead of when the event happens!

Every time a user interacts with a web page, such as clicking on a link, pressing a key, or moving a mouse, an event occurs that our program can detect and then respond to.

Every time a button/page/element is clicked, a callback function is called. So the program can continue processing the rest of the code while it is waiting for the click event to happen. 

        document.body.addEventListener("click", doSomething);
        
        In the example above, the event listener has been added to the documents 
        body element. It will call the function doSomething() when any part of the 
        page is clicked on

Types of Events

Mouse Events

click event that occurs when a mouse button is clicked, the mousedown and mouseup events occur before a click event is fired,dblclick event(do not use click and double click on the same element),

Keyboard Events

Three events that occur when a user presses a key are: keydown, keypress and keyup. When a user presses a key, the events occur in that order.

The keydown event occurs when a key is pressed and will continue to occur if the key is held down.The keypress event occurs after a keydown event but before a keyup event. The keypress event only occurs for keys that produce character input (plus the 'Delete' key). This means that itâ€™s the most reliable way to find out the character that was pressed on the keyboard. The keyup event occurs when a key is released.

To understand the differences in these events, it is important to distinguish between a physical key on the keyboard and a character that appears on the screen. The keydown event is the action of pressing a key, whereas the keypress event is the action of a character being typed on the screen.

Touch Events

Many modern devices now support touch events. These are used on smartphones and tablets, as well as touch-screen monitors, satellite navigators and trackpads.

The touchstart event occurs when a user initially touches the surface, The click event can be thought of as a "tap" in the context of a touch event. (has delay of 300ms),The touchend event occurs when a user stops touching the surface, The touchmove event occurs after a user has touched the screen then moves around without leaving,

There are no 'swipe' events. These need to be created by using a combination of touchstart, touchmove, and touchleave events that monitor the distance and direction moved from start to finish of a touch event

Stopping Default Behavior

Some elements have default behavior associated with certain events. For example, when a user clicks on a link, the browser redirects to the address in the href attribute and a form is submitted when the user clicks on the Submit button. preventDefault() is a method of the event object that can be used inside the callback function to stop the default behavior happening.

Event Propagation

When you click on an element, you are actually clicking on all the elements it is nested inside of.

If you click on one of the -li- elements, you are also clicking on the -ul-, -body- and -html- elements. An event is said to propagate as it moves from one element to another. Event propagation is the order that the events fire on each element. There are two forms of event propagation: bubbling and capturing.

Bubbling is when the event fires on the element clicked on first, then bubbles up the document tree, firing an event on each parent element until it reaches the root node.Capturing starts by firing an event on the root element, then propagates downwards, firing an event on each child element until it reaches the target element that was clicked on.

Third argument of the addEventListener() function - Capturing

Capturing: The addEventListener() method has a third parameter, which is a boolean value that specifies whether capturing should be used or not. It defaults to false, which is why bubbling happens by default. To implement capturing,add the third argument as a true boolean variable.

                        console.log('Clicked on ul'),true);

                        console.log('Clicked on li'),true);

Event Delegation and Its main Application-use

Event delegation can be used to attach an event listener to a parent element in order to capture events that are triggered by its child elements.

A better way is to attach the event listener to the parent -ul- element, that has many li elements is to use the target property to identify the element that was clicked on
