Objects

const and let

The keyword const is used when the variable will not be reassigned to another value, whereas let is used if the variable might be reassigned later in the program. Variables have to be declared before they can be used.

If the variable references a non-primitive data type, such as an array, function or object, then using const will not make it immutable. This means the underlying data inside the object can change (known as mutating the object).
Even if you use const to declare a variable, non-primitive data types can still be mutated later in the program.

Variable Scope

Using const and let to declare variables means they are block scoped, so their value only exists inside the block they are declared in.

Global Scope

Any variable declared outside of a block is said to have global scope. This means it is accessible everywhere in the program. While this may seem to be a good idea at first, it is not considered good practice.
A variable having a global scope means its available inside and outside the other lower level scopes,but local variables in these scopes have a higher precedence than the global variables.
Defining a variable in a scope without using let or const makes that variable global and can be accessed anywhere. { a = 10 }

Local Scope

Blocks '{ --this space here-- }' can be used to create a local scope. This means that any variables defined inside a block using the let or const will only be available inside that block and not be accessible outside of that bloc
If let or const are not used, the variable will have global scope and be available outside the block.

Template Literals

Template Literals allows the use both types of quote mark within the string and interpolation of JavaScript code. This means that a JavaScript expression can be inserted inside a string and the result will be displayed.
`She said, "It's Me!"`
const age = 39;
`I will be ${ age + 1 } next year`;

Arrays

Arrays, sets, and maps are data structures used in JavaScript to store lists of values. The main array methods include Pop, Push, Shift, and Unshift

Deleting an array item

The delete operator will remove an item from an array
delete avengers[3];
The value that was in position 3 has will be deleted from the array, but the space that it occupied will still there and contain a value of undefined.
This means the array still has the same number of elements, and the position can still be referenced as an index, but it will just return undefined:
To complete;y delete the item and its position use array methods such as pop and shift.

Destructuring Arrays

Destructuring an array is the concept of taking values out of an array and presenting them as individual values.
Destructuring allows us to assign multiple values at the same time, using arrays:
const [x, y] = [1, 2];
x = 1 and y = 2 outside the array

Maps

They are a convenient way of keeping a list of key and value pairs, and are similar to 'hashes', or 'hash tables' or 'dictionaries' in other programming languages.
The set() method is be used to add a key and value pair to a map. The first value is the key and the second is the value:
romanNumerals.set(1,'I');
A map is a bit like a dictionary where you can look up a value based on the key. To look up a value, we can use the get() method:
romanNumerals.get(4); //'IV'

Difference Between Parameters and Arguments

Parameters and arguments are terms that are often used interchangeably to represent values provided for the function as an input.
There is a subtle difference though: any parameters a function needs are set when the function is defined. When a function is invoked, it is provided with arguments.

If a parameter is not provided as an argument when the function is invoked, the function will still be invoked, but the parameter will be given a value of undefined.
The function returns NaN because the function cannot do the required operation with undefined.

If too many arguments are provided when a function is invoked, the function will work as normal and the extra arguments will be ignored.

Functions that accept any number of arguments

The rest operator ...--argument-- can be used to make a function accept any number of arguments that it is given 

                              function mean(...values) {
                                 let total = 0;
                                 for(const value of values) {
                                     total += value;
                                 }
                                 return total/values.length;
                             }

                              mean(2,8,13); // 7.6666...
                              mean(2,8);  // 5

A Callback Function

A function that is passed as an argument to another is known as a callback. Functions can be used as arguments because they are first class objects that behave like any other JS object. 

                              function sing(song,callback) {
                                  console.log(`I'm singing along to ${song}.`);
                                  callback();
                              }

The callback is provided as a parameter, then invoked inside the body of the function.
Note that the callback dance is passed as an argument without parentheses. This is because the argument is only a reference to the function. The actual callback is invoked in the body of the function, where parentheses are used.

Callback Arrow function/anonymous function

This is the most common way in which call back functions are used 

                           function sing(song) {
                              console.log(`I'm singing along to ${song}.`);
                           }

                           //linking a callback function to a function call
                           sing('Let It Go',()=>{ 
                              console.log("I'm standing on my head.");
                           });

Sorting Numbers in JS Functions

Numbers in JS are sorted alphabetically.Hence to sort them correctly, the following callback functions are used. 

                           function numerically(a,b){
                              return a-b;
                           }

                           //for very large numbers,to remove another error that JS makes
                           function numerically2 (a,b) {
                              if (a < b) {
                                 return -1;
                              } else if (a> b) {
                                 return 1;
                              } else {
                                 return 0;
                              }
                            }

                            //Calling the functions
                            const numbers =[1,3,12,5,23,18,7];
                            console.log(numbers.sort(numerically));
                            console.log(numbers.sort(numerically2));
